PROJECT SPECIFICATION: LazyMise
=========================================================
Author       : Mochammad Tora Sifa
Methodology  : Prototype / Iterative (V1 -> V2 -> V3)
Tech Stack   : Golang, Bubble Tea (TUI), Go TOML Parser, os/exec
Description  : A lazygit-inspired Terminal User Interface (TUI) for the mise-en-place (mise) polyglot version manager. Built to minimize friction when switching/adding language versions without leaving the terminal or manually editing TOML files.

=========================================================
CORE PHILOSOPHY (The Golden Rules)
=========================================================
1. NO STAGING AREA: Changes are Auto-Saved to the .mise.toml file instantly. We don't want users losing config changes if they accidentally close the app.
2. INSTANT WRITE, BACKGROUND EXECUTE: When a user selects a tool, write to the TOML file immediately (1ms). The actual `mise install` download MUST happen in the background using Go Goroutines so it doesn't freeze the UI.
3. CONTEXT AWARE: The app must know if it's in a project folder (Local .mise.toml) or home directory (Global ~/.mise.toml) and act accordingly.
4. SAFETY vs SPEED: Use pop-ups for safety, but provide power-user shortcuts to bypass them.

=========================================================
ARCHITECTURE & DATA FLOW
=========================================================
1. INIT: App starts -> Detects Context (Global vs Local) -> Parses .mise.toml into a Go Struct.
2. RENDER: Bubble Tea reads the Go Struct -> Draws the UI list.
3. INTERACT: User presses keys -> App updates the Go Struct in memory -> App instantly writes struct to .mise.toml file -> App spawns `go exec.Command("mise", "install")` in background -> UI updates status to [Installing...].
4. EXIT: User presses 'q' -> App terminates. (If downloads are running, they continue in the OS background because we used Goroutines, not attached processes).

=========================================================
KEYBINDINGS SPEC
=========================================================
[Global Navigation]
- Up/k    : Move up
- Down/j  : Move down
- q       : Quit/Go back

[Main Dashboard View (Tool List)]
- Enter   : Open Version Picker (V2). For V1, just prompt for version string.
- a       : Add a new tool (Auto-saves to TOML).
- d       : Delete selected tool (Auto-saves to TOML).
- x       : Force run `mise install` for everything manually.
- e       : Open Environment Variables panel (V3).

[Version Picker View (V2)]
- Enter       : Select version. If not downloaded -> Y/N Pop-up -> Download in background.
- Shift+Enter : Select version -> FORCE download instantly (No pop-up).
- /           : Search/Filter available versions.
- Esc         : Go back to Dashboard.
